# Claude Fleet Adoption Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Adopt the `origin/codex/cleanup` fleet (5 agents / 26 skills, content-complete), collapse it to a directly-edited `.claude/` single source for Claude Code only, reshape the roster to 6 agents (merge `observer`+`scribe` → `sre-steward`; reinstate `researcher` and `prompt-engineer`), port the sister repo's fail-closed allowlist guard onto `sre`/`sre-steward`, graft the sister repo's post-import improvements, and rewrite the root docs.

**Architecture:** The fleet content already exists on `origin/codex/cleanup` in a `canonical/` → `generated/{claude,copilot}` projection. We merge that branch (codex tree wins; this branch's `docs/` is kept), promote the *generated Claude projection* to be the directly-edited source under `.claude/`, delete the generator and all Copilot artifacts, then apply roster/guard/graft/docs deltas as separately reviewable commits.

**Tech Stack:** Claude Code agents/skills (Markdown + YAML frontmatter), Python 3 stdlib scripts, Git Bash on Windows.

**Spec:** `docs/superpowers/specs/2026-07-17-claude-fleet-adoption-design.md` (approved). Prior evidence: `docs/superpowers/specs/2026-07-13-copilot-fleet-redesign-design.md`, `docs/AUDIT-2026-07-12.md` (both restored on this branch).

## Global Constraints

- **Branch:** all work happens on `research-items`; each task ends in its own commit; push after the final task only (earlier pushes optional).
- **Windows interpreter rule:** invoke Python as `python` (never `python3` — on this machine it's the Microsoft Store stub). `scripts/gate_a.py` self-solves via `sys.executable`.
- **Git Bash path mangling:** `git show rev:.claude/...` mangles the colon-path. Use `git show 'rev:path'` only for paths NOT starting with a dot; otherwise use `git checkout <rev> -- <path>` or `MSYS_NO_PATHCONV=1 git show ...`.
- **Description limit:** every agent/skill `description` ≤ 1,024 characters.
- **No `model:` pins** on any agent (fleet inherits the session model).
- **No new `hooks:` on plugin-context assumptions:** frontmatter `PreToolUse` hooks are valid for *project-scope* agents (probed fact recorded in the sister guard's docstring) — this repo's agents are project-scope, so frontmatter wiring is correct here.
- **Verbatim-move discipline:** when relocating existing prose (agent bodies, skill sections), move it with git/file operations or copy-paste — never retype from memory. Where this plan says "move section X", the section is identified by its heading; take its bytes as-is unless an edit is explicitly listed.
- **Out of scope (do not do, even if adjacent):** eval-harness rebuild beyond mechanical path fixes; CI redesign; distribution/plugin packaging; renaming skills that survive in place.
- **Generated-header rule:** every file taken from `generated/claude/` starts with a `# GENERATED by scripts/generate_fleet.py ...` comment line inside the frontmatter. That line must be deleted in every adopted file (the generator is being removed; the files become source).

- [ ] **Step 3: Write the launcher `scripts/readonly-guard-hook.sh`**

```sh
#!/bin/sh
# Fail-closed launcher for the read-only allowlist guard (scripts/readonly-guard.py).
# Wired via per-agent frontmatter PreToolUse hooks (matcher: Bash) on sre and sre-steward.
# Protocol: guard exits 42 = allow (empty stdout), 43 = deny (JSON on stdout).
# If NO interpreter answers with the guard's own exit codes, the guard is missing/broken:
# DENY — these hooks only fire for guarded agents, so failing closed cannot hit the user's session.
IN=$(cat)
G="${CLAUDE_PROJECT_DIR:-.}/scripts/readonly-guard.py"
for C in python py python3; do
  command -v "$C" >/dev/null 2>&1 || continue
  OUT=$(printf '%s' "$IN" | "$C" -I -S "$G" 2>/dev/null); RC=$?
  if [ "$RC" -eq 42 ]; then exit 0; fi
  if [ "$RC" -eq 43 ]; then printf '%s' "$OUT"; exit 0; fi
done
printf '%s' '{"hookSpecificOutput":{"hookEventName":"PreToolUse","permissionDecision":"deny","permissionDecisionReason":"Read-only guard unavailable: no interpreter answered with the guard exit codes (tried python, py, python3). Bash is denied for guarded agents until the guard is restored."}}'
exit 0
```
